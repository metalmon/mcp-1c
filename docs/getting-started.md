# Быстрый старт

Подключаем AI-ассистента к вашей базе 1С за 10 минут.

## Что это и зачем

**Проблема:** Вы пишете код на BSL в 1С. Современные AI-модели (YandexGPT, GigaChat, Claude и др.) умеют генерировать код, но не знают структуру *вашей* конфигурации — какие справочники, документы, реквизиты у вас есть. Без этого AI выдумывает несуществующие реквизиты и пишет нерабочий код.

**Решение:** mcp-1c — мост между AI-ассистентом и вашей базой 1С. После установки AI видит:
- Все объекты конфигурации (справочники, документы, регистры, ...)
- Реквизиты, табличные части, измерения, ресурсы
- Код модулей (через поиск по выгрузке конфигурации)
- Структуру форм
- Данные через запросы (только SELECT)

AI не просто «знает 1С вообще» — он знает *именно вашу* конфигурацию и пишет точный код под неё.

### Что такое MCP

MCP (Model Context Protocol) — открытый протокол, который позволяет AI-ассистентам обращаться к внешним источникам данных. Это стандарт, который поддерживают десятки AI-клиентов: [Claude Desktop](https://claude.ai/download), [Cursor](https://cursor.com), [Windsurf](https://codeium.com/windsurf) и другие. Полный список — на [modelcontextprotocol.io](https://modelcontextprotocol.io).

Вам не нужно разбираться в деталях протокола — просто установите mcp-1c и AI-ассистент получит доступ к вашей базе.

### Как это работает

```
┌─────────────────┐         ┌──────────────┐         ┌──────────────┐
│  AI-ассистент   │  stdio  │  mcp-1c.exe  │  HTTP   │  База 1С     │
│  (любой MCP-    │ ──────► │  (посредник) │ ──────► │  (расширение │
│   совместимый)  │ ◄────── │              │ ◄────── │  HTTP-сервис)│
└─────────────────┘         └──────────────┘         └──────────────┘
```

1. Вы задаёте вопрос AI про вашу конфигурацию
2. AI через mcp-1c запрашивает нужные метаданные из 1С
3. Получает реальную структуру и пишет код, который подходит именно вашей базе

## Требования

- **1С:Предприятие 8.3 или 8.5** (коммерческая или учебная версия)
- **AI-клиент с поддержкой MCP** — например [Claude Desktop](https://claude.ai/download), [Cursor](https://cursor.com), [Windsurf](https://windsurf.com), [VS Code](https://code.visualstudio.com) и другие (полный список — [ниже](#шаг-4-настроить-ai-клиент))
- **Windows** или **Linux** (на macOS HTTP-сервис 1С [не поддерживается](https://www.online-ufa.ru/content/articles/limit-1c-for-macos/), но mcp-1c и автоустановка работают)

> Учебную версию 1С можно скачать бесплатно: [uc1.1c.ru](https://uc1.1c.ru/uchebnaya-versiya-1s/)

## Установка

### Шаг 1. Скачать mcp-1c

Скачайте `mcp-1c.exe` (Windows) или `mcp-1c` (macOS/Linux) со страницы [Releases](../../releases).

> Или соберите из исходников: `go build -o mcp-1c ./cmd/mcp-1c/`

### Шаг 2. Установить расширение в базу 1С

Расширение добавляет в вашу базу HTTP-сервис, через который mcp-1c получает данные. Оно **не меняет** вашу конфигурацию — это именно расширение.

**Автоматически (рекомендуется):**

```bash
# Windows
mcp-1c --install "C:\путь\к\вашей\базе"

# macOS / Linux
mcp-1c --install ~/Documents/InfoBase
```

Программа сама найдёт установленную платформу 1С, загрузит расширение и обновит конфигурацию БД.

> Если платформа установлена в нестандартную папку, укажите путь явно:
> `mcp-1c --install "путь_к_базе" --platform "/custom/path/to/1cv8"`
>
> Если версия платформы не определяется автоматически (нестандартный путь без номера версии), укажите её явно:
> `mcp-1c --install "путь_к_базе" --platform "/custom/path/to/1cv8" --platform-version 8.3.13`

> Если в базе настроена аутентификация для Конфигуратора (DESIGNER), укажите учётные данные:
> `mcp-1c --install "путь_к_базе" --db-user Admin --db-password пароль`

> **Клиент-серверная база (MS SQL, PostgreSQL).** Если база зарегистрирована на сервере 1С, добавьте флаг `--server` и укажите в `--install` строку подключения в формате `сервер\имя_базы`:
> ```bash
> mcp-1c --install "srv-1c\buh_prod" --server --db-user Admin --db-password pass
> ```

> Перед установкой закройте все сеансы 1С (Конфигуратор, Предприятие) для этой базы.

**Вручную (если автоматическая не сработала):**

1. Соберите .cfe из исходников (см. [Сборка расширения](../README.md#сборка-расширения-из-исходников))
2. Откройте базу в **Конфигураторе**
3. Меню **Конфигурация → Расширения конфигурации**
4. Нажмите **«Добавить из файла»** и выберите собранный `MCP_HTTPService.cfe`
5. Нажмите **F7** (обновить конфигурацию базы данных)

### Шаг 3. Запустить HTTP-сервис 1С

Чтобы mcp-1c мог обращаться к базе, нужно опубликовать HTTP-сервис 1С. Есть два способа:

#### Способ 1. Публикация через Apache или IIS (рекомендуется)

Стандартный способ публикации HTTP-сервисов 1С. Работает на **Windows** (IIS или Apache) и **Linux** (Apache или ibsrv). Подходит как для разработки, так и для постоянной работы.

1. Откройте базу в **Конфигураторе от администратора**
2. **Администрирование → Публикация на веб-сервере**
3. Укажите имя (например `base`), отметьте `MCPService`
4. Нажмите **Опубликовать**

URL в этом случае: `http://localhost/base/hs/mcp-1c`

#### Способ 2. Встроенный HTTP-сервер /HTTPPort (только Windows, для быстрого запуска)

Параметр `/HTTPPort` запускает встроенный HTTP-сервер прямо в процессе 1С. Это **недокументированный** параметр платформы, удобный для быстрой проверки и разработки. Для постоянной работы рекомендуется публикация через Apache или IIS (способ 1).

> **Ограничения:** `/HTTPPort` работает **только на Windows**. На Linux и macOS этот параметр недоступен.

**Windows (коммерческая версия):**
```cmd
"C:\Program Files\1cv8\8.3.XX.XXXX\bin\1cv8.exe" ENTERPRISE /F "C:\путь\к\базе" /HTTPPort 8080
```

**Windows (учебная версия):**
```cmd
"C:\Program Files\1cv8t\8.5.XX.XXXX\bin\1cv8t.exe" ENTERPRISE /F "C:\путь\к\базе" /HTTPPort 8080
```

> Замените `XX.XXXX` на вашу версию платформы. Посмотреть можно в папке `C:\Program Files\1cv8\` или `C:\Program Files\1cv8t\`.

Окно 1С откроется и будет работать как HTTP-сервер. **Не закрывайте его** — пока оно открыто, HTTP-сервис доступен.

#### Платформенные ограничения

**macOS:** Платформа 1С [не поддерживает HTTP-сервисы на macOS](https://www.online-ufa.ru/content/articles/limit-1c-for-macos/). Используйте Windows (можно в виртуальной машине, например UTM).

**Linux:** Используйте публикацию через Apache или автономный сервер ibsrv (способ 1). Встроенный HTTP-сервер (`/HTTPPort`) на Linux недоступен.

#### Проверка

Откройте в браузере или выполните в терминале:

```bash
# Если публикация через Apache/IIS (способ 1):
curl http://localhost/base/hs/mcp-1c/version

# Если встроенный HTTP-сервер /HTTPPort (способ 2):
curl http://localhost:8080/hs/mcp-1c/version
```

Ожидаемый ответ:
```json
{"version":"0.4.0"}
```

Если видите ответ — HTTP-сервис 1С работает, переходите к шагу 4.

#### Claude Desktop

[Claude Desktop](https://claude.ai/download) — десктопное приложение от Anthropic.

**Файл конфигурации:**

- **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`
- **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`

Если файла нет — создайте его. Добавьте:

```json
{
  "mcpServers": {
    "1c": {
      "command": "C:\\путь\\к\\mcp-1c.exe",
      "args": ["--base", "http://localhost:8080/hs/mcp-1c"]
    }
  }
}
```

**Перезапустите** Claude Desktop (полностью закройте и откройте заново).

> **Режим доступа инструментов.**
> По умолчанию Claude Desktop загружает инструменты «по запросу» (Auto) — при большом количестве подключённых серверов это экономит контекст, но может приводить к тому, что часть инструментов mcp-1c не обнаруживается при поиске. Если вы используете только mcp-1c (или 1-2 коннектора), рекомендуем переключить режим на **Always Available**:
>
> **«+» → Connectors → Tool access → Always available**
>
> В этом режиме все 9 инструментов загружаются сразу и AI видит их целиком.

#### Claude Code (CLI)

[Claude Code](https://docs.anthropic.com/en/docs/claude-code) — консольный AI-ассистент от Anthropic. Работает в терминале.

Добавьте MCP-сервер командой:

**Windows:**
```cmd
claude mcp add 1c --transport stdio -- cmd /c C:\путь\к\mcp-1c.exe --base http://localhost:8080/hs/mcp-1c
```

**macOS / Linux:**
```bash
claude mcp add 1c --transport stdio -- /путь/к/mcp-1c --base http://localhost:8080/hs/mcp-1c
```

> По умолчанию сервер добавляется в текущий проект. Чтобы сделать его доступным во всех проектах, добавьте `--scope user`:
> ```bash
> claude mcp add 1c --scope user --transport stdio -- /путь/к/mcp-1c --base http://localhost:8080/hs/mcp-1c
> ```

Проверить, что сервер добавлен:

```bash
claude mcp list
```

Удалить сервер:

```bash
claude mcp remove 1c
```

#### Cursor

[Cursor](https://cursor.com) — IDE с встроенным AI на базе VS Code.

**Файл конфигурации:**

- **Для конкретного проекта:** `.cursor/mcp.json` в корне проекта
- **Глобально (для всех проектов):** `~/.cursor/mcp.json`

Создайте файл и добавьте:

```json
{
  "mcpServers": {
    "1c": {
      "command": "C:\\путь\\к\\mcp-1c.exe",
      "args": ["--base", "http://localhost:8080/hs/mcp-1c"]
    }
  }
}
```

Или добавьте через интерфейс: **Cursor Settings → Tools & MCP → Add new MCP Server**.

Если в 1С включена аутентификация HTTP-сервиса, добавьте `--user` и `--password` в `args`:

```json
{
  "mcpServers": {
    "1c": {
      "command": "C:\\путь\\к\\mcp-1c.exe",
      "args": [
        "--base", "http://localhost:8080/hs/mcp-1c",
        "--user", "mcp",
        "--password", "пароль"
      ]
    }
  }
}
```

Для локальной разработки через `Makefile`:

```bash
make cursor-install CURSOR_SERVER_NAME=1c-business CURSOR_TOOLSET=business CURSOR_PROFILE=auto MCP_USER=mcp MCP_PASSWORD=пароль
```

Для прод-окружений вместо `--password` в JSON используйте `env` и передавайте секрет через переменные окружения (`MCP_1C_USER`, `MCP_1C_PASSWORD`).

> **Ограничение:** Cursor оптимально работает при общем количестве до ~40 MCP-инструментов по всем серверам. У mcp-1c всего 9 инструментов, так что проблем не будет, но если у вас подключено много других серверов — отключайте лишние.

#### VS Code + GitHub Copilot

[VS Code](https://code.visualstudio.com) с расширением [GitHub Copilot](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot) поддерживает MCP-серверы начиная с версии VS Code 1.99.

> MCP-инструменты работают только в режиме **Agent** в чате Copilot. Убедитесь, что выбран Agent, а не Chat или Edit.

**Файл конфигурации:**

- **Для конкретного проекта:** `.vscode/mcp.json` в корне проекта
- **Глобально:** выполните команду **MCP: Open User Configuration** в Command Palette (`Ctrl+Shift+P`)

Создайте файл `.vscode/mcp.json` и добавьте:

```json
{
  "servers": {
    "1c": {
      "type": "stdio",
      "command": "C:\\путь\\к\\mcp-1c.exe",
      "args": ["--base", "http://localhost:8080/hs/mcp-1c"]
    }
  }
}
```

> **Формат отличается от Claude Desktop:** ключ верхнего уровня — `"servers"` (не `"mcpServers"`), и для каждого сервера нужно указать `"type": "stdio"`.

Или добавьте через интерфейс: **Command Palette → MCP: Add Server → stdio**.

**Конфигурация через JSON:**

```json
{
  "mcpServers": {
    "1c": {
      "command": "C:\\путь\\к\\mcp-1c.exe",
      "args": ["--base", "http://localhost:8080/hs/mcp-1c"]
    }
  }
}
```

> MCP-инструменты в Continue работают только в режиме **Agent**. Переключитесь на него в интерфейсе чата.

> Если вы уже настроили mcp-1c для Claude Desktop — скопируйте конфигурацию в `.continue/mcpServers/` и Continue подхватит её автоматически.

Напишите AI-ассистенту:

AI вызовет инструмент `get_metadata_tree` и покажет список справочников, документов и регистров вашей базы.

## Поиск по коду конфигурации (--dump)

Инструмент `search_code` работает по локальной выгрузке конфигурации в файлы. Чтобы включить его:

1. Выгрузите конфигурацию через Конфигуратор: **Конфигурация -> Выгрузить конфигурацию в файлы**
2. Добавьте флаг `--dump` с путём к выгрузке:

```json
{
  "mcpServers": {
    "1c": {
      "command": "C:\\путь\\к\\mcp-1c.exe",
      "args": [
        "--base", "http://localhost:8080/hs/mcp-1c",
        "--dump", "C:\\путь\\к\\выгрузке"
      ]
    }
  }
}
```

Без флага `--dump` инструмент `search_code` не будет доступен.

### Режимы поиска

`search_code` поддерживает три режима:

| Режим | Описание |
|-------|---------|
| `smart` | Полнотекстовый поиск с BM25-ранжированием (по умолчанию). Результаты отсортированы по релевантности. |
| `regex` | Регулярное выражение (синтаксис Go regexp). |
| `exact` | Точная подстрока, регистронезависимый поиск. |

**BSL-синонимы.** В режиме `smart` работает автоматическое расширение запроса: поиск по английским именам функций находит русские и наоборот. Например, `StrFind` найдёт модули с `СтрНайти`, а `Процедура` — модули с `Procedure`.

**Фильтрация.** Можно ограничить поиск по типу метаданных (`category`: Документ, Справочник, ОбщийМодуль и т.д.) и типу модуля (`module`: МодульОбъекта, МодульМенеджера, МодульФормы и т.д.).

### Неблокирующий запуск

При старте с `--dump` MCP-сервер доступен сразу — индекс строится в фоне. Пока индексация не завершена, `search_code` вернёт сообщение «search index is building, please retry». Для конфигураций с 13 000+ модулей первая индексация занимает ~7 секунд (шардированная параллельная сборка по числу ядер CPU).

### Кеширование индекса

Построенный индекс автоматически сохраняется на диск. При следующих запусках загружается из кеша — старт практически мгновенный.

**Расположение кеша:**

| ОС | Путь |
|----|------|
| macOS | `~/Library/Caches/mcp-1c/` |
| Linux | `~/.cache/mcp-1c/` |
| Windows | `%LocalAppData%\mcp-1c\` |

Размер кеша зависит от конфигурации и обычно составляет ~100-200 МБ для крупных конфигураций.

### Принудительная перестройка индекса (--reindex)

Флаг `--reindex` заставляет mcp-1c заново построить поисковый индекс, игнорируя кеш. Используйте его, когда:

- Вы обновили выгрузку конфигурации (выполнили новый `DumpConfigToFiles`)
- Индекс работает некорректно или повреждён

```json
{
  "mcpServers": {
    "1c": {
      "command": "C:\\путь\\к\\mcp-1c.exe",
      "args": [
        "--base", "http://localhost:8080/hs/mcp-1c",
        "--dump", "C:\\путь\\к\\выгрузке",
        "--reindex"
      ]
    }
  }
}
```

> После перестройки индекса флаг `--reindex` можно убрать — при следующих запусках будет использоваться обновлённый кеш.

## Аутентификация

Если в базе 1С настроена аутентификация для HTTP-сервисов, добавьте параметры `--user` и `--password`. Примеры ниже показаны для формата Claude Desktop / Cursor / Windsurf (`"mcpServers"`). Для VS Code + Copilot используйте формат `"servers"` (см. [выше](#vs-code--github-copilot)).

**Через аргументы:**

```json
{
  "mcpServers": {
    "1c": {
      "command": "C:\\путь\\к\\mcp-1c.exe",
      "args": [
        "--base", "http://localhost:8080/hs/mcp-1c",
        "--user", "Admin",
        "--password", "пароль"
      ]
    }
  }
}
```

**Через переменные окружения:**

```json
{
  "mcpServers": {
    "1c": {
      "command": "C:\\путь\\к\\mcp-1c.exe",
      "args": ["--base", "http://localhost:8080/hs/mcp-1c"],
      "env": {
        "MCP_1C_USER": "Admin",
        "MCP_1C_PASSWORD": "пароль"
      }
    }
  }
}
```

**Claude Code (CLI):**

```bash
claude mcp add 1c --transport stdio -e MCP_1C_USER=Admin -e MCP_1C_PASSWORD=пароль -- /путь/к/mcp-1c --base http://localhost:8080/hs/mcp-1c
```

## Ограничения MCP-клиентов

Разные AI-клиенты по-разному работают с MCP-инструментами. У mcp-1c всего 9 инструментов, поэтому перечисленные ограничения обычно не создают проблем — но полезно знать о них, если вы подключаете несколько MCP-серверов.

| Клиент | Ограничение | Рекомендация |
|--------|------------|--------------|
| **Claude Desktop** | В режиме Auto загружает инструменты «по запросу» и может не найти часть из них | Включите **Always Available** (см. [инструкцию](#claude-desktop)) |
| **Cursor** | Оптимально до ~40 инструментов суммарно по всем серверам | Отключайте серверы, которые не используете |
| **Windsurf** | Лимит 100 инструментов суммарно по всем серверам | Управляйте через иконку MCPs в панели Cascade |
| **VS Code + Copilot** | MCP работает только в режиме Agent | Переключите режим чата на Agent |
| **Continue** | MCP работает только в режиме Agent | Переключите режим чата на Agent |
| **JetBrains** | Требуется подписка AI Assistant | Включите режим Codebase в чате AI Assistant |

## Устранение неполадок

**«1C platform not found» при --install**
Платформа 1С не найдена в стандартных папках. Укажите путь явно: `mcp-1c --install "путь" --platform "/path/to/1cv8"`. Если версия не определяется из пути, добавьте `--platform-version 8.3.13` (укажите вашу версию). Минимальная поддерживаемая версия: 8.3.10.

**«Файл не найден» при --install**
Проверьте путь к базе. Это папка, в которой лежит файл `1Cv8.1CD`.

**«connection refused» при curl**
HTTP-сервис 1С не запущен. Если используете `/HTTPPort` — убедитесь, что окно 1С открыто. Если публикацию через Apache/IIS — проверьте, что веб-сервер запущен и публикация активна.

**«404» при обращении к /hs/mcp-1c/...**
- Расширение не установлено или не активно. Проверьте: Конфигуратор → Расширения → галочка «Активно» напротив MCP_HTTPService.
- Конфигурация БД не обновлена. Нажмите F7 в Конфигураторе.

**«address already in use»**
Порт 8080 занят. Используйте другой: `/HTTPPort 9090` и обновите URL в конфиге AI-клиента.

**AI не видит инструменты 1С**
- Убедитесь, что AI-клиент перезапущен после изменения конфига.
- Проверьте JSON на ошибки (лишние/недостающие запятые, неэкранированные слеши).
- Убедитесь, что путь к `mcp-1c.exe` правильный и полный (абсолютный, не относительный).

**AI не находит нужный инструмент (например get_object_structure)**
- В Claude Desktop: **«+» → Connectors → Tool access → Always available**. В режиме Auto (по умолчанию) Claude загружает инструменты «по запросу» и может не найти нужный.
- В Cursor / Windsurf: проверьте, что сервер 1c включён и все его инструменты активны в настройках MCP.

**MCP-сервер не запускается в VS Code + Copilot**
- Убедитесь, что VS Code версии 1.99 или выше.
- Убедитесь, что в чате Copilot выбран режим **Agent** (не Chat и не Edit).
- Проверьте формат: в `.vscode/mcp.json` ключ верхнего уровня — `"servers"` (не `"mcpServers"`), и для каждого сервера указан `"type": "stdio"`.

**MCP-сервер не работает в JetBrains IDE**
- Убедитесь, что IDE версии 2025.1 или выше.
- Проверьте: **Settings → Tools → AI Assistant → Model Context Protocol (MCP)** — сервер должен быть в списке и включён.
- Если вы импортировали конфигурацию из Claude Desktop — включите опцию **Automatically enable new and changed MCP servers**.

**«Достигнуто ограничение учебной версии» / «Предельное количество подключений»**
- Учебная версия 1С ограничивает количество одновременных подключений. Закройте лишние сеансы (Конфигуратор, другие окна 1С). mcp-1c закрывает HTTP-соединения после каждого запроса, чтобы минимизировать эту проблему.
