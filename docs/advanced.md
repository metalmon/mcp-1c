# Платные версии mcp-1c

Открытая версия mcp-1c дает AI-ассистенту доступ к вашей базе 1С: 9 инструментов, метаданные, поиск по коду, запросы к данным. Для многих задач этого хватает.

Но в ежедневной работе есть то, что базовыми инструментами не решается. Запрос тормозит, в расширениях баг, после обновления платформы что-то отвалилось, нужно переписать модальные вызовы на асинхронные. На каждую такую задачу уходит полдня.

Расширенная версия добавляет 15 инструментов для этих случаев. Профессиональная версия идет дальше: семантический поиск, граф зависимостей, массовый анализ кода, аудит безопасности, метрики кодовой базы и помощник по обновлению типовых.

## 15 инструментов Расширенной версии

### 1. Оптимизатор запросов

Анализирует запрос 1С на 15 антипаттернов производительности. Русский и английский синтаксис BSL.

Что находит:
- Соединения по неиндексированным полям
- Использование `*` вместо конкретных полей
- Неоптимальные подзапросы вместо временных таблиц
- Избыточные параметры виртуальных таблиц
- Функции в условиях WHERE, отключающие индексы
- Отсутствие РАЗРЕШЕННЫЕ при программном выполнении
- N+1 запросы в цикле

Пример:

```
> Оптимизируй этот запрос:
>
> ВЫБРАТЬ * ИЗ РегистрНакопления.ОстаткиТоваровНаСкладах.Остатки()
> ЛЕВОЕ СОЕДИНЕНИЕ Справочник.Номенклатура
> ПО ОстаткиТоваровНаСкладахОстатки.Номенклатура = Номенклатура.Ссылка
```

AI-ассистент вызовет оптимизатор и покажет: (1) `ВЫБРАТЬ *` читает все поля, включая ненужные, замените на конкретные; (2) виртуальная таблица Остатки() без параметров отборов делает полный скан; (3) добавьте РАЗРЕШЕННЫЕ, если запрос выполняется программно.

Не заменяет полноценный аудит, но типичные проблемы ловит за секунды.

### 2. Анализ кода

Рентген модуля: какие процедуры вызываются (включая рекурсию), от каких объектов метаданных зависит модуль, где побочные эффекты (запись, удаление, проведение). Все запросы с номерами строк, количество экспортных методов, уровень вложенности. Не "объясни что делает", а структурированные данные.

```
> Проанализируй модуль объекта документа ЗаказКлиента
```

AI-ассистент получит данные по модулю и расскажет: логика, зависимости, потенциальные проблемы.

### 3. Проверка кода BSL (линтер)

Автоматический контроль качества: 30 диагностик с кодами правил (BSL001-BSL030), уровнями серьезности, конкретными рекомендациями. Работает мгновенно, прямо внутри MCP-сервера.

Что ловит (выборочно):

**Транзакции:** незакрытая НачатьТранзакцию без пары, транзакция без Попытка/Исключение, забытый ОтменитьТранзакцию() в блоке Исключение, вложенные транзакции без проверки ТранзакцияАктивна().

**Производительность:** Запрос.Выполнить() внутри цикла, ПолучитьОбъект() / НайтиПоКоду() внутри цикла, конкатенация строк в цикле вместо СтрСоединить().

**Безопасность:** Выполнить() с динамическим кодом, конкатенация строк в тексте запроса вместо УстановитьПараметр(), запрос с условием ГДЕ без параметров.

**Обработка ошибок:** пустой блок Исключение, запись нового объекта без Попытка, незакрытое соединение (HTTP, FTP, COM), ЭтотОбъект.Записать() внутри ПередЗаписью/ОбработкаПроведения (бесконечная рекурсия).

**Сложность и структура:** процедуры длиннее 200 строк, больше 7 параметров, глубокая вложенность, устаревшие методы (Вопрос(), ОткрытьФормуМодально(), ПолучитьФорму()), неиспользуемые переменные и мертвый код после Возврат.

```
> Проверь модуль объекта документа РеализацияТоваровУслуг
```

Можно фильтровать по серьезности (только ошибки, ошибки + предупреждения, все) и отключать отдельные правила.

Анализ кода - чтобы понять. Проверка кода BSL - чтобы проверить. Используются в паре.

### 4. Справочник синтаксиса платформы

Около 10 000 определений методов, свойств и типов. Данные берутся из .hbk-файлов документации платформы, тех самых, что использует Синтакс-помощник в Конфигураторе. Полнотекстовый поиск по всему справочнику.

```
> Какие методы есть у ТаблицаЗначений для поиска строк?
```

AI-ассистент найдет НайтиСтроки, Найти с описанием параметров, типов и примерами. Не галлюцинация, а точные данные из документации 1С. Работает без Конфигуратора, без EDT, без интернета.

### 5. Чтение кода расширений

Чтение исходного кода из скомпилированных расширений (.cfe) живой базы. Модули объектов, модули менеджеров, модули форм, общие модули - все читается прямо из .cfe файла, без выгрузки через Конфигуратор.

```
> Покажи код модуля объекта из расширения МоёРасширение
```

### 6. Поиск по расширениям

Поиск по всем расширениям базы одновременно, с пометкой [Расш] в результатах, чтобы было видно, откуда пришел код. Когда в базе 10+ расширений от разных разработчиков, без этого отладка конфликтов превращается в мучение.

```
> Найди все обработчики ПриЗаписи в расширениях
```

### 7. Совместимость версий

Обновление платформы - отдельная боль. Этот инструмент держит базу знаний по breaking changes между версиями. При обновлении покажет, какие методы стали deprecated, что переименовали, что убрали, и предложит конкретные замены.

```
> Проверь совместимость моего кода с платформой 8.3.25
```

AI-ассистент проанализирует код модуля, сверит с базой изменений: "строка 47: МодальныйРежим убран в 8.3.25, замените на ОписаниеОповещения". Поддерживает матрицу совместимости для нескольких версий одновременно.

### 8. Конвертер модальных вызовов

Автоматический рефакторинг синхронных конструкций в асинхронные:

| Было | Стало |
|------|-------|
| `Вопрос(...)` | `ПоказатьВопрос(Оповещение, ...)` |
| `Предупреждение(...)` | `ПоказатьПредупреждение(Оповещение, ...)` |
| `ВвестиСтроку(...)` | `ПоказатьВводСтроки(Оповещение, ...)` |
| `ВвестиЧисло(...)` | `ПоказатьВводЧисла(Оповещение, ...)` |
| `ВвестиДату(...)` | `ПоказатьВводДаты(Оповещение, ...)` |
| `ПоместитьФайл(...)` | `НачатьПомещениеФайла(Оповещение, ...)` |
| `ПолучитьФайл(...)` | `НачатьПолучениеФайла(Оповещение, ...)` |

```
> Переведи модуль формы ЗаказКлиента на асинхронные вызовы
```

AI-ассистент найдет все модальные вызовы, создаст процедуры обработки оповещений и переструктурирует логику. Для перехода на веб-клиент и мобильное приложение (8.3.18+) - фактически обязательный шаг.

### 9. Генератор запросов

Описание на русском языке - на выходе запрос 1С с учетом виртуальных таблиц регистров и реальных метаданных вашей конфигурации.

```
> Покажи остатки товаров на складе Основной на начало месяца
```

AI-ассистент запросит структуру регистра ОстаткиТоваровНаСкладах, узнает имена измерений и ресурсов и сгенерирует запрос:

```bsl
ВЫБРАТЬ
    ОстаткиОстатки.Номенклатура.Наименование КАК Товар,
    ОстаткиОстатки.КоличествоОстаток КАК Остаток
ИЗ
    РегистрНакопления.ОстаткиТоваровНаСкладах.Остатки(
        &НачалоМесяца,
        Склад = &Склад
    ) КАК ОстаткиОстатки
```

Работает и для сложных запросов с несколькими таблицами.

### 10. Генератор печатных форм

Описание на естественном языке - на выходе код модуля печати и XML-макет табличного документа.

```
> Сделай печатную форму для документа Счет с шапкой (номер, дата, контрагент),
> таблицей товаров (номенклатура, количество, цена, сумма) и итогом
```

Получите процедуру `Печать()` с заполнением параметров, XML-макет с областями (Шапка, СтрокаТаблицы, Подвал) и регистрацию команды печати. Печатные формы - одна из самых рутинных вещей в 1С. Каждый раз одно и то же: макет, параметры, код заполнения. Наконец-то можно делегировать.

### 11. Мультибазовость

Одновременная работа с несколькими базами 1С:

- Сравнить структуру метаданных между dev и prod
- Искать код одновременно в нескольких конфигурациях
- Работать с тестовой и рабочей базой в одном сеансе

```bash
mcp-1c-advanced --base dev=http://localhost:8081/hs/mcp-1c --base prod=http://localhost:8082/hs/mcp-1c
```

```
> Сравни структуру справочника Контрагенты в dev и prod
```

### 12. Long polling (работа без Apache/IIS)

В стандартном режиме mcp-1c требует HTTP-сервис 1С - либо встроенный HTTP-сервер (Windows), либо публикацию через Apache/IIS. Для многих это серьезный порог входа.

Long polling переворачивает схему. Не mcp-1c стучится в 1С, а 1С сама стучится в Go-сервер за заданиями. Расширение периодически опрашивает mcp-1c, получает запрос, выполняет, возвращает результат. Итого: расширение в базе + бинарник mcp-1c - и все работает. Без Apache, без IIS, без публикации HTTP-сервисов.

### 13. Песочница кода

Безопасное выполнение фрагментов BSL в контексте вашей базы. Не путать с полным `execute_code` (который есть в некоторых MCP-серверах и выполняет что угодно) - тут работают ограничения:

- Черный список опасных операций (удаление объектов, транзакции, мета-исполнение, внешние обработки)
- Защита от обхода блокировок через гомоглифы и zero-width символы
- Ограничение размера кода (10 КБ)
- Обязательное подтверждение пользователем перед выполнением в полном режиме
- Режим "только чтение" по умолчанию

```
> Выполни: Сообщить(Справочники.Контрагенты.НайтиПоНаименованию("ООО Ромашка"))
```

### 14. Шаблоны промптов

Заготовки для типовых задач: заполнение табличной части по данным регистра, запрос с виртуальными таблицами, обработка проведения с контролем остатков, настройка прав доступа, код обмена данными. AI-клиент видит эти шаблоны и использует как стартовую точку. Не нужно каждый раз формулировать задачу заново.

### 15. Умный контекст

Сжатие метаданных и выборочная подача. Вместо полного дерева метаданных (для ERP это сотни килобайт) модель получает только релевантную часть. Спрашиваете про РеализацияТоваровУслуг - AI-ассистент получит структуру этого документа и связанных регистров. Не всю конфигурацию. На практике: больше места в контекстном окне для вашего кода, меньше "забывания" в длинных диалогах.

## Безопасность

**Локальный процесс.** MCP-1C работает на вашей машине. Общается только с вашей базой 1С по локальной сети. Никаких внешних серверов для работы инструментов не требуется.

**Локальные модели.** При использовании Ollama, LM Studio или llama.cpp данные не покидают вашу инфраструктуру. Подходит для закрытых контуров и работы с конфиденциальными данными (зарплата, персональные данные, финансовая отчетность).

**Песочница.** Все операции, изменяющие данные, требуют явного подтверждения. 8 паттернов блокировки опасных конструкций, включая защиту от Unicode-омоглифов.

**Аудит.** Каждое действие пишется в журнал (JSON Lines).

**Офлайн-режим.** Работает без интернета до 72 часов после последней проверки лицензии. Для работы на объектах с нестабильным подключением - необходимость.

## Планы: Профессиональная и Корпоративная

Проектируем еще два тарифа. Пока в разработке, сроки не называем.

**Профессиональная** (для команд) - все из Расширенной плюс семантический поиск по смыслу (на базе векторных индексов), граф зависимостей для impact analysis, массовый анализ кода по всей конфигурации, расширенный BSL аудит с SARIF-отчетами для CI/CD, метрики кодовой базы (техдолг, сложность, тренды), помощник по обновлению типовых (diff между релизами + ваши доработки = список конфликтов) и сравнение конфигураций.

**Корпоративная** - все из Профессиональной плюс Trust Layer (маскирование персональных данных перед отправкой в модель: фамилии, ИНН, телефоны заменяются на плейсхолдеры), развертывание в Docker/K8s, мультипользовательский режим, ролевые профили доступа, поддержка с SLA.

Сначала стабилизируем Расширенную.

### AI-клиенты и модели

Все 24 инструмента (9 базовых + 15 расширенных) работают с любой моделью. MCP-1C не привязан к конкретной нейросети, не знает и не определяет, какая модель работает на стороне клиента.

| | Примеры |
|---|---|
| **Локальные модели** | Ollama, LM Studio, llama.cpp - данные не покидают вашу сеть |
| **Облачные сервисы** | Claude, GPT, YandexGPT, GigaChat |
| **IDE с AI** | Cursor, VS Code (Continue, Cline, Copilot), Windsurf, JetBrains IDEs |

Какую модель использовать - решаете вы. Для работы с конфиденциальными данными 1С подходят локальные модели: весь обмен происходит на вашей машине, ничего не уходит наружу.
